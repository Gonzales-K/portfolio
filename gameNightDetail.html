The Log-in Page

The landing page when one visits this site is the log-in page. This application utilizes responsive design techniques to dynamically render to fit the viewport of the device being used.

The navigation bar conditionally renders based on if there is a valid authentication token. If there is no token, the user is only presented with links to the pages we allow anyone to view. This allows us to use one navigation bar element for all pages of the application.

The Log-in element renders as a card centered on the screen. It has input fields for username and password as well as a button for submitting the credentials and a link to create a new account. The password input field is obscured for security purposes. Upon submission, the application sends a request to the server to authenticate the user. If authentication fails the user is prompted to try again or create an account. If successful, they are redirected to the games page.

The create account link routes the user to the registration page.

The Registration Page

The registration page is a form that allows users to create an account. There are input fields for various data-types.

The password input field is obscured for security purposes. Upon submission, using the create account button, the program will validate all data. If all data meets requirements, the password will be encrypted and the hashed password will be stored in the database. The user will then be routed to the games page.

The location field pulls all locations from the database and allows the user to select one. If the user's location is not in the database, there is an Add Location button that will bring up a modular form to create a new location. This will add to the database and add it to the dropdown location options.

The Games Page

The games page will display all games in the database. The functionality of this page is similar to the my games, events, and my events pages with the exception that the events page displays events rather than games and the my games/events pages only display what the user has either created or subscribed to.

Each game is displayed as a card within a flexbox. As games are added to the database they will be displayed dynamically adding rows as needed and displaying up to three columns.
Each card is comprised of an image, title, a button for viewing game details and a dropdown to add it to the user's favorites or watch lists.

The search bar will filter the games displayed based on title.
